0Overview & design constraints

A production-grade, touchless, deterministic engine that estimates the equity value of an Indian MSME by pricing it against comparable listed companies, grounded on a single data source (Dun & Bradstreet dnbhoovers).

core = Python stdlib only NO LLM in the pipeline NO network in core runs on a mock returning the REAL D&B schema

Single data source

All identity, industry, financials and directors come from four D&B datasources. No second provider.

Mock now, live last

The live swap is one method body (an httpx.post); response paths are identical, so nothing downstream changes.

Deterministic core

Discovery + valuation are pure functions. Seeded synthetic universe. Same input → same output.

Never fake precision

Every valuation is a range with a disclosed EV basis, a confidence label and a full audit trail. If nothing is computable → "none".

Money units

D&B returns INR Thousand; converted to INR Crore (÷10,000) at the normalization boundary. All math in Crore.

Decoupled

core/ never imports mock_api/; the client is injected in run.py.

2End-to-end pipeline

Seven stages from a company name to a ranged equity valuation. Every stage appends to the audit trail.

1

Resolve

company_search → best DUNS by highest confidenceCode.

in: name, "IN" · out: DUNS · audit: TARGET_RESOLVED / NO_MATCH
2

Fetch & normalize

company_information + company_financials (+ management) → Company dataclass; every monetary field ÷10,000 to Crore; EBIT, growth, margin derived.

out: Company (Crore) · listing from CIN L/U · market EV from marketData
3

Economic profile (rule-based)

Classifies operating_model / value_chain / customer_type from activities text. The collision rule (a manufacturing VERB, not the noun "manufacturers") is the LLM-swap seam.

out: EconomicProfile · audit: TARGET_PROFILED
4

Data-quality gate

Weighted per-field grade (A–D) + valuable flag. If not valuable → stops with insufficient_data, no fabricated number.

out: DataQuality · audit: DATA_QUALITY_GRADE
5

Discover peers

Hard mismatch filter → weighted 5-dimension similarity (0–1) over the whole universe; return all ranked, take top 15.

out: ranked peers + rejected (with reasons) · audit: PEER_REJECTED, PEERS_RANKED
6

Value (multi-method)

Similarity-weighted trading multiples → Tukey trim → quality positioning → EV → less net debt → less DLOM → equity range. Three methods for triangulation.

out: Valuation (low/mid/high) + market cross-check · audit: PEERS_WEIGHTED, METHOD_COMPUTED, HEADLINE_SELECTED
7

Confidence + outputs

Six-signal confidence (incl. triangulation agreement) → result.json + dashboard.html.

out: confidence + breakdown, full audit · audit: CONFIDENCE_SCORED, COMPLETE

4Calculation deep-dive

4.1 Peer similarity (discovery)

Candidates first pass a hard mismatch filter (operating_model value_chainmajor_industry must all match — else rejected before scoring). Survivors score 0–1 as a weighted sum:

# dimension = raw × weight industry 0.40 = 1.0 same NAICS-3 subsector · 0.6 same Hoovers · else 0 scale 0.20 = 1 / (1 + |log1p(rev_t) − log1p(rev_p)|) margin 0.15 = max(0, 1 − 5 × |margin_t − margin_p|) customer 0.15 = 1.0 if same customer_type else 0 export 0.10 = 1.0 if same is_exporter else 0.3 score = Σ(dimension)

4.2 Similarity weighting — inexact peers count less (v1.5.0)

Not every peer is an exact match. Each peer's multiple is down-weighted by its match quality, then multiples are aggregated with a weighted percentile — so a thin or loose peer set cannot distort the headline.

weight = clamp( (score − 0.40) / (0.85 − 0.40), floor 0.15, cap 1.0 ) score ≥ 0.85 → 1.00 (full match) score 0.625 → 0.50 score ≤ 0.40 → 0.15 effective_peer_count = Σ weight # drives range-widening + confidence coverage

4.3 Quality positioning & the multiple

Rather than the flat peer median, the central multiple is read at the target's quality position — its EBITDA-margin percentile in the peer set — so a weaker-margin company earns a lower multiple.

q_rank = percentile-rank of target EBITDA margin among peers mid_q = clamp(q_rank, 0.15, 0.85) window = 0.20 (0.30 if effective<10) low_q,high_q = clamp(mid_q ∓ window, 0.05, 0.95) multiple_x = weighted_percentile(trimmed peer multiples, x_q) # Tukey 1.5×IQR trim first

4.4 EV → equity bridge

EV_peer = market_ev (listed comp) else capital_employed (book, only if <3 listed) implied_EV = positioned_multiple × target_driver # EBITDA / Revenue / EBIT net_debt = debt − cash DLOM = 0.00 if listed · 0.30 if rev<100 · 0.25 if rev<500 · else 0.20 equity = (implied_EV − net_debt) × (1 − DLOM) # low / mid / high

Headline = first computable of EV/EBITDA → EV/Revenue → EV/EBIT; all three reported for triangulation. For a listed target, the headline equity is cross-checked against its own market cap.

4.5 Confidence — six discriminating signals (v1.4.0)

score = 0.20·profile + 0.20·min(eff_peers,15)/15 + 0.10·(EBITDA>0) + 0.10·(methods/3) + 0.25·triangulation_agreement + 0.15·comparable_tightness triangulation = 1 − clamp((max_mid/min_mid − 1)/0.80, 0, 1) tightness = 1 − clamp(CV(multiples)/0.45, 0, 1) label HIGH ≥ 0.75 · MEDIUM ≥ 0.50 · else LOW
Not saturated. Across the universe confidence spans 0.31–0.95 (the 5 wrong entities score LOW); a target whose methods diverge scores lower even with 15 peers.

8Trust & controls (touchless)

Structured audit trail

Every D&B call, rejected peer and valuation decision is a typed, time-stamped, ordered record. DECISION rows are the judgement calls.

Data-quality gate

Grades A–D and can halt with insufficient_data before any number is produced.

Discriminating confidence

Reflects triangulation agreement and comparable tightness — it earns its value.

Disclosed EV basis

Market vs book, and the effective peer count, are always disclosed.

Market cross-check

Listed targets are validated against their own market cap (±25% acceptance gate).

Anti-overfitting gate

The universe backtest is an acceptance check — the build fails if positioning stops generalizing.
